# DESIGN.md

Дизайн-система фронтенда (apps/web). Реализация — `apps/web/src/styles.css`. Направление:
**тёплый графит + янтарный акцент**, моноширинный для машинных данных. Тёмная тема по умолчанию.

## Сцена

Супер-админ в полутёмной комнате смотрит на нагрузку CPU-only сервера в 2 ночи; разработчик днём
выпускает ключ из IDE. Тёмная база, тёплый нейтраль (не сине-чёрный), высокий, но не режущий контраст.

## Цвет (OKLCH, токены в `:root`)

Нейтраль подкрашен в тёплый (hue ~70–75, chroma ~0.01). Никаких чистых #000/#fff.

| Токен | Значение | Роль |
|---|---|---|
| `--bg` | `oklch(0.165 0.012 72)` | фон приложения |
| `--surface` | `oklch(0.205 0.012 72)` | панели, таблицы |
| `--surface-2` | `oklch(0.245 0.013 72)` | hover, вложенные блоки |
| `--border` | `oklch(0.30 0.012 72)` | разделители |
| `--text` | `oklch(0.94 0.008 75)` | основной текст |
| `--text-muted` | `oklch(0.68 0.012 75)` | вторичный |
| `--accent` | `oklch(0.80 0.145 75)` | янтарь: primary action, бренд, выделение |
| `--accent-ink` | `oklch(0.20 0.02 75)` | текст на янтарном |
| `--ok` | `oklch(0.74 0.15 150)` | approved/успех |
| `--warn` | `oklch(0.82 0.14 75)` | pending/ожидание (янтарный родственник) |
| `--err` | `oklch(0.65 0.20 25)` | failed/revoked/ошибка |
| `--live` | `oklch(0.76 0.13 215)` | живые события, пульс |

Стратегия: **Restrained** — янтарь только на действия/выделение/бренд (≤10% поверхности),
семантика — на статусы. Светлый режим (`[data-theme="light"]`) — бумага+чернила, тот же hue.

## Типографика

- UI: `Inter, system-ui, sans-serif`. Данные/ключи/тайминги/метрики: `"JetBrains Mono", ui-monospace, monospace`.
- Шкала ratio 1.2, фиксированная (rem): 12, 13, 14(base), 16, 19, 23, 28, 34.
- Вес: 400 body, 500 labels/кнопки, 600 заголовки. Контраст веса, не размера, для плотного UI.
- Числа — `font-variant-numeric: tabular-nums` везде в таблицах/метриках.

## Сетка и ритм

- Шелл: фиксированный sidebar (нав) + контент. Sidebar чуть холоднее/темнее контента (`--surface`).
- Spacing (rem): 0.25 / 0.5 / 0.75 / 1 / 1.5 / 2 / 3. Варьировать, не одинаковый padding везде.
- Радиусы: 6px контролы, 10px панели, 999px badge/pill.
- Контент-ширина прозы 70ch; таблицы во всю ширину.

## Компоненты (состояния обязательны: default/hover/focus/active/disabled/loading/error)

- **Кнопка:** primary (янтарь, `--accent-ink`), ghost (прозрачная, border), danger (контур `--err`). focus-ring 2px `--accent` со смещением.
- **Поле ввода:** `--surface`, border, focus-ring янтарный. Ошибка — border `--err` + сообщение.
- **Badge статуса:** pill, фон = семантика @ ~15% (color-mix), текст = семантика. PENDING=warn, APPROVED=ok, REVOKED=err. Точка-индикатор слева.
- **Таблица:** плотная, зебра отсутствует, разделители `--border`, sticky-заголовок, hover-строка `--surface-2`, моно для id/токенов/чисел.
- **Метрика:** НЕ hero-template. Малый caps-лейбл + крупное моно-число + дельта/спарклайн. В ряд по сетке, разного веса.
- **Графики:** лёгкий SVG (бар/линия/спарклайн), без либ. Цвета: requests=`--accent`, errors=`--err`, latency=`--live`.
- **Live-индикатор:** мягкий пульс точки `--live` (opacity, не layout), 2s ease-in-out.

## Движение

150–250ms, ease-out (quart/expo). Только состояние: hover, focus, появление строки, copy-feedback,
обновление метрики (короткий тон подсветки). Никаких оркестраций загрузки. Новые live-строки — fade+translateY 6px.

## Копирайт

Без воды, без тире/«--». Кнопки — глаголы. Пустые состояния учат интерфейсу.
Момент показа сырого ключа — явное предупреждение «ключ больше не будет показан» + copy.
